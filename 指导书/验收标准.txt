/hello 路由	执行 curl -i http://127.0.0.1:8080/hello	返回 200 OK 和正确响应正文
用户路由	执行 curl -i http://127.0.0.1:8080/users/zhangsan	返回 200 OK 和张三的用户信息
404处理	执行 curl -i http://127.0.0.1:8080/missing	返回 404 Not Found
多请求处理	同时或连续发送多个HTTP请求	每个请求均能获得完整HTTP响应，服务器不中断
请求数量控制	请求总数达到 max_requests	服务器完成当前请求后正常退出
连接清理	观察响应完成和客户端断开后的处理	执行 EPOLL_CTL_DEL + close (client_fd)，无连接泄漏
epoll代码检查	学生现场指出事件循环关键代码	使用 epoll_create1()、epoll_ctl()、epoll_wait()、LT+EPOLLIN
技术限制检查	检查服务端源代码	未使用 select、多线程、多进程或线程池
自动测试	运行项目测试脚本	两个路由、404、请求计数和关闭清理全部通过
文件5：建议学生完成顺序
建议学生按以下顺序完成：

先完成实训内容2，确保3个客户端连接、消息分帧和退出清理正确。

复制或复用 EpollServer 的事件循环。

将文本消息处理函数替换为 HTTP 请求处理函数。

增加课件规定的两个路由、404 响应和关闭清理。

使用 curl -i 验证状态码、响应头和正文。

总结： 实训内容2解决“怎样用 epoll 管理多个 TCP 连接”，WebServer V1.0解决“连接就绪后怎样正确处理 HTTP 请求”。
